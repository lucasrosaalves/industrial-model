# Generated Client Usage

This guide explains how to use a package generated by
`industrial_model generate`. The examples use a Cognite Core package generated
with:

```bash
industrial_model generate \
  --token "$CDF_TOKEN" \
  --project my-cdf-project \
  --base-url https://westeurope-1.cognitedata.com \
  --data-model cdf_cdm/CogniteCore/v1 \
  --output ./cognite_core_client \
  --client-name CogniteCoreClient
```

The generated package contains a facade class, one typed sub-client per view,
typed filters, typed property literals, writable models, and aggregation models.

---

## Client Construction

Generated facades embed the data model id used during generation, so application
code does not need to pass `DataModelId` again.

### User Token

Use this when you have a bearer token and want the generated client to create
the `Engine` internally:

```python
from cognite_core_client import CogniteCoreClient

client = CogniteCoreClient(
    user_token="<bearer-token>",
    project="my-cdf-project",
)
```

When the package is generated from a standard CDF URL such as
`https://westeurope-1.cognitedata.com`, the facade stores
`DEFAULT_CLUSTER = "westeurope-1"`. Override it only when needed:

```python
client = CogniteCoreClient(
    user_token="<bearer-token>",
    project="my-cdf-project",
    cluster="az-phx-001",
)
```

### Existing CogniteClient

Use this when your application already owns authentication and SDK
configuration:

```python
from cognite.client import ClientConfig, CogniteClient
from cognite.client.credentials import Token
from cognite_core_client import CogniteCoreClient

cognite_client = CogniteClient(
    ClientConfig(
        client_name="my-app",
        project="my-cdf-project",
        credentials=Token("<bearer-token>"),
        cluster="westeurope-1",
    )
)

client = CogniteCoreClient(cognite_client)
```

### Existing Engine

Use this when you already created an `industrial_model.Engine`:

```python
from cognite.client import ClientConfig, CogniteClient
from cognite.client.credentials import Token
from industrial_model import DataModelId, Engine
from cognite_core_client import CogniteCoreClient

cognite_client = CogniteClient(
    ClientConfig(
        client_name="my-app",
        project="my-cdf-project",
        credentials=Token("<bearer-token>"),
        cluster="westeurope-1",
    )
)

engine = Engine(
    cognite_client=cognite_client,
    data_model_id=DataModelId(
        space="cdf_cdm",
        external_id="CogniteCore",
        version="v1",
    ),
)

client = CogniteCoreClient(engine)
```

## Facade And View Clients

Each CDF view becomes one attribute on the facade:

```python
from cognite_core_client import CogniteCoreClient

client = CogniteCoreClient(user_token="<bearer-token>", project="my-cdf-project")

client.cognite_asset  # CogniteAssetClient
client.cognite_equipment  # CogniteEquipmentClient
client.cognite_time_series  # CogniteTimeSeriesClient
client.cognite_file  # CogniteFileClient
client.cognite_unit  # CogniteUnitClient
```

Each view client exposes:

| Method | Purpose |
|--------|---------|
| `query()` | Fetch one page of typed model instances |
| `query_all_pages()` | Fetch all pages into a plain list |
| `search()` | Full-text search with optional filters |
| `aggregate()` | Count, min, max, sum, or average |
| `upsert()` | Insert or update writable models |
| `delete()` | Delete model instances |
| `*_async()` | Async equivalent of each method |

---

## Query Basics

`query()` returns one `PaginatedResult[T]`.

```python
page = client.cognite_asset.query(
    filters={
        "space": {"eq": "my-instance-space"},
        "name": {"prefix": "Pump"},
    },
    include=["parent", "type"],
    limit=100,
)

assets = page.data
next_cursor = page.next_cursor
```

Continue with the cursor:

```python
next_page = client.cognite_asset.query(
    filters={"name": {"prefix": "Pump"}},
    cursor=page.next_cursor,
    limit=100,
)
```

Fetch every page:

```python
assets = client.cognite_asset.query_all_pages(
    filters={"tags": {"containsAny": ["critical", "rotating"]}},
    include=["parent"],
    limit=500,
)
```

`include` controls relation loading. Relation fields not included are excluded
from the generated query to avoid unnecessary traversal.

## Filter Operators

Filter types are generated per view. For `CogniteAsset`, the generated
`CogniteAssetFilter` contains keys such as `name`, `description`, `tags`,
`parent`, `root`, `type`, `assetClass`, `space`, `externalId`, `AND`, `OR`, and
`NOT`.

Common scalar operators:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "externalId": {"prefix": "pump-"},
        "name": {"eq": "Pump 001"},
        "description": {"exists": True},
        "space": {"in_": ["production", "staging"]},
    }
)
```

List operators:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "tags": {"containsAny": ["critical", "safety"]},
        "aliases": {"containsAll": ["pump", "feed"]},
    }
)
```

Datetime and boolean filters appear on views that expose those property types:

```python
from datetime import UTC, datetime

time_series = client.cognite_time_series.query_all_pages(
    filters={
        "isStep": {"eq": False},
        "sourceUpdatedTime": {
            "gte": datetime(2025, 1, 1, tzinfo=UTC),
        },
    }
)
```

Relation ids use `InstanceId`:

```python
from industrial_model import InstanceId

assets = client.cognite_asset.query_all_pages(
    filters={
        "parent": {
            "eq": InstanceId(space="my-instance-space", external_id="plant-001")
        }
    }
)
```

## Boolean Queries

Use `AND`, `OR`, and `NOT` to compose filters.

### AND

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "AND": [
            {"space": {"eq": "production"}},
            {"name": {"prefix": "Pump"}},
            {"description": {"exists": True}},
        ]
    }
)
```

### OR

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "OR": [
            {"name": {"prefix": "Pump"}},
            {"name": {"prefix": "Compressor"}},
            {"tags": {"containsAny": ["rotating"]}},
        ]
    }
)
```

### NOT

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "AND": [
            {"space": {"eq": "production"}},
            {"NOT": {"tags": {"containsAny": ["archived", "disabled"]}}},
        ]
    }
)
```

### NOR-style logic

There is no separate `NOR` key. Express NOR as `NOT` around an `OR` block:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "NOT": {
            "OR": [
                {"tags": {"containsAny": ["archived"]}},
                {"name": {"prefix": "TEST-"}},
                {"space": {"eq": "sandbox"}},
            ]
        }
    }
)
```

### Mixed boolean logic

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "AND": [
            {
                "OR": [
                    {"type": {"code": {"eq": "pump"}}},
                    {"type": {"code": {"eq": "compressor"}}},
                ]
            },
            {"NOT": {"tags": {"containsAny": ["archived"]}}},
            {"space": {"eq": "production"}},
        ]
    },
    include=["type", "parent"],
)
```

## Nested Relation Queries

Generated relation filters accept either an `InstanceId` filter or the related
view's typed filter when the relation points to another generated view.

Filter assets by asset type properties:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "type": {
            "code": {"eq": "pump"},
            "standard": {"eq": "ISO"},
        }
    },
    include=["type"],
)
```

Filter assets by parent asset properties:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "parent": {
            "name": {"prefix": "Area A"},
            "space": {"eq": "production"},
        }
    },
    include=["parent"],
)
```

Nested filters can be combined with boolean operators:

```python
assets = client.cognite_asset.query_all_pages(
    filters={
        "AND": [
            {
                "parent": {
                    "OR": [
                        {"name": {"prefix": "Area A"}},
                        {"name": {"prefix": "Area B"}},
                    ]
                }
            },
            {
                "type": {
                    "NOT": {
                        "OR": [
                            {"code": {"eq": "temporary"}},
                            {"code": {"eq": "test"}},
                        ]
                    }
                }
            },
        ]
    },
    include=["parent", "type"],
)
```

List relation properties use list-specific filters:

```python
from industrial_model import InstanceId

time_series = client.cognite_time_series.query_all_pages(
    filters={
        "assets": {
            "containsAny": [
                InstanceId(space="production", external_id="pump-001"),
                InstanceId(space="production", external_id="pump-002"),
            ]
        }
    },
    include=["assets"],
)
```

## Search

`search()` runs full-text search over generated query properties and can be
combined with filters.

```python
assets = client.cognite_asset.search(
    query="feed pump",
    query_properties=["name", "description", "aliases"],
    query_operator="AND",
    filters={
        "AND": [
            {"space": {"eq": "production"}},
            {"NOT": {"tags": {"containsAny": ["archived"]}}},
        ]
    },
    limit=25,
)
```

Search equipment by name or description:

```python
equipment = client.cognite_equipment.search(
    query="compressor",
    query_properties=["name", "description"],
    filters={"space": {"eq": "production"}},
    limit=50,
)
```

## Aggregations

`aggregate()` supports `count`, `avg`, `min`, `max`, and `sum`.

Count all matching assets:

```python
counts = client.cognite_asset.aggregate(
    filters={"space": {"eq": "production"}},
    aggregate_type="count",
)
```

Group counts by type:

```python
by_type = client.cognite_asset.aggregate(
    filters={"space": {"eq": "production"}},
    aggregate_type="count",
    group_by_properties=["type"],
    limit=100,
)
```

Aggregate numeric properties on views that expose them. For example,
`Cognite360ImageAnnotation` exposes `confidence`:

```python
confidence = client.cognite_360_image_annotation.aggregate(
    aggregate_type="avg",
    aggregation_property="confidence",
    group_by_properties=["formatVersion"],
    limit=100,
)
```

## Mutations

Generated models inherit from `WritableViewInstance`, so they always include
`external_id` and `space`.

### Upsert

```python
from industrial_model import InstanceId
from cognite_core_client.models import CogniteAsset

asset = CogniteAsset(
    external_id="pump-001",
    space="production",
    name="Pump 001",
    description="Main feed pump",
    tags=["critical", "rotating"],
    parent=InstanceId(space="production", external_id="plant-001"),
)

client.cognite_asset.upsert([asset])
```

Use `replace=True` to replace properties instead of patching them:

```python
client.cognite_asset.upsert([asset], replace=True)
```

Upsert related instances by using generated models or `InstanceId` values:

```python
from cognite_core_client.models import CogniteEquipment

equipment = CogniteEquipment(
    external_id="pump-001-motor",
    space="production",
    name="Pump 001 motor",
    asset=InstanceId(space="production", external_id="pump-001"),
)

client.cognite_equipment.upsert([equipment])
```

### Delete

```python
client.cognite_asset.delete(
    [CogniteAsset(external_id="pump-001", space="production")]
)
```

## Async Usage

Every generated view client has async equivalents:

```python
page = await client.cognite_asset.query_async(
    filters={"name": {"eq": "Pump 001"}},
    include=["parent"],
)

assets = await client.cognite_asset.query_all_pages_async(limit=500)

matches = await client.cognite_asset.search_async(
    query="pump",
    query_properties=["name", "description"],
)

counts = await client.cognite_asset.aggregate_async(aggregate_type="count")

await client.cognite_asset.upsert_async([asset])
await client.cognite_asset.delete_async([asset])
```

Do not call synchronous generated-client methods from inside an async event loop.
Use the `_async` variants instead.
